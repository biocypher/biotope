# biotope

CLI for the BioCypher ecosystem. Turns Croissant-described data into a BioCypher knowledge graph; tracks the metadata in a git-like workflow.

!!! warning "Pre-alpha, developer-facing"

    APIs, CLI flags, and config-file layouts will change. End-user docs come after the design stabilises.

## Quick start: from `init` to a knowledge graph

```bash
uv pip install biotope

# 1. Scaffold a project.
biotope init my-kg --purpose "What approved drugs target genes in T2D?"
cd my-kg

# 2. Declare what the graph must contain (agent-friendly flags).
biotope map --entity gene --entity disease --entity drug \
            --relation gene_associated_with_disease

# 3. Bring in data + its Croissant metadata.
biotope add data/ot.parquet --license CC-BY-4.0

# 4. Generate an unresolved mapping scaffold for that Croissant file.
#    The scaffold has one slot per declared entity/relation plus an inspector
#    appendix listing record sets, field kinds, and sample rows.
biotope map scaffold .biotope/datasets/ot.jsonld

# 5. Resolve the slots ‚Äî pick a record set, fields, transforms for each entity
#    and relation. Two equivalent paths:
biotope map                   # interactive guided wizard (humans)
# ‚Ä¶or edit mappings/*.mapping.yaml directly and consult:
biotope map inspect .biotope/datasets/ot.jsonld --json
biotope map preview --json    # status + projected schema + sample tuples

# 6. Build a runnable BioCypher project. Strict: rejects unresolved slots.
biotope build
biotope view
```

All semantic decisions (which record set, which fields, which transforms) are made by the human or copilot agent. biotope only enumerates options, validates, and previews ‚Äî it never auto-picks a "best" record set.

## Commands

### Project lifecycle

- `biotope init` ‚Äî scaffold a project (`.biotope/`, `AGENTS.md`, `project.yaml`, `git init`).

### Semantic mapping

- `biotope map` ‚Äî bare command. If any intent flag (`--purpose`, `--entity`, `--relation`, `--source`, `--notes`, `--clear-*`, `--show`) is passed, it updates `project.yaml` non-interactively. Otherwise it launches the guided wizard.
- `biotope map inspect <croissant>` ‚Äî deterministic field catalogue + sample rows. `--json` for agents.
- `biotope map scaffold <croissant>` ‚Äî emit an unresolved mapping scaffold with an inspector comment appendix.
- `biotope map preview [<mapping>]` ‚Äî validate a (partial) mapping; show projected BioCypher schema + sample tuples. `--json` for agents.

### Git-like metadata VCS

- `biotope add` ‚Äî stage data files; baker enriches the Croissant entry under `.biotope/datasets/`.
- `biotope mv` ‚Äî move tracked files; updates metadata paths.
- `biotope status` ‚Äî show staged/modified files and validation state.
- `biotope commit` ‚Äî commit metadata changes.
- `biotope log` ‚Äî show metadata commit history.
- `biotope push` / `biotope pull` ‚Äî sync metadata with a remote.
- `biotope check-data` ‚Äî verify data files against recorded checksums.

### Knowledge-graph construction

- `biotope discover` ‚Äî rank registered adapters and local Croissant files against `required_entities`.
- `biotope propose-alignment` ‚Äî propose cross-mapping `same_node` equivalences.
- `biotope build` ‚Äî materialise a runnable BioCypher project from mappings + alignment. Emits `config/schema_config.yaml` (with `namespace` and autogenerated `input_label`) and per-mapping generated Python under `build/generated/<stem>/`.
- `biotope view` ‚Äî node/edge counts for the most recent build.
- `biotope benchmark` ‚Äî quality/coverage metrics (skeleton in v1).

### Deprecated

- `biotope describe` ‚Äî removed; folded into `biotope map` intent flags.
- `biotope propose-mapping` ‚Äî deprecated alias for `biotope map scaffold`. The old heuristic ("one RecordSet per node type, FK fields as edges") is gone; the alias now produces an unresolved scaffold for human/agent completion.

### Promises (not feature-complete)

- `biotope read` ‚Äî NLP ingestion + health-check entry.
- `biotope chat` ‚Äî provider-agnostic conversational interface (biochatter backend ships first).
- `biotope search` / `biotope get` / `biotope annotate` / `biotope config` ‚Äî auxiliary surfaces.
