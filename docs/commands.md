# Commands

Overview of `biotope` CLI verbs by pipeline stage. For flags and examples, see the [per-command reference](api-docs/init.md).

## Project lifecycle

- `biotope init` ‚Äî scaffold a project (`.biotope/`, `project.yaml`, `git init`).

## Data acquisition + tracking

- `biotope get <url>` ‚Äî download a file (optionally into `--output-dir`) and, unless `--no-add`, track it.
- `biotope add <path>` ‚Äî stage data files or rooted directories; baker writes the Croissant entry under `.biotope/datasets/`. `--derived-from` records provenance for human/agent-extracted derivatives. For curated metadata that doesn't fit as CLI flags (descriptions, citations, per‚Äìrecord-set fields), `add` also drops a `.biotope.yaml` scaffold next to the dataset ‚Äî review it, then run `biotope annotate apply <dir>` to merge it into the manifest.
- `biotope mv` / `biotope rm` ‚Äî move or untrack files and update metadata paths.
- `biotope queue` ‚Äî show every dataset grouped by pipeline state (`raw` / `processed` / `mapped`). The recommended dashboard during a build.
- `biotope mark <dataset> <status>` ‚Äî manually set a dataset's `biotope:status`.

## Semantic mapping

- `biotope map` ‚Äî bare command. If any intent flag (`--purpose`, `--entity`, `--relation`, `--source`, `--notes`, `--clear-*`, `--show`) is passed, it updates `project.yaml` non-interactively. Otherwise it launches the guided wizard.
- `biotope map inspect <croissant>` ‚Äî deterministic field catalogue + sample rows. `--json` for agents.
- `biotope map scaffold <croissant>` ‚Äî emit an unresolved mapping scaffold with an inspector comment appendix.
- `biotope map preview [<mapping>]` ‚Äî validate a (partial) mapping; show projected BioCypher schema + sample tuples. `--json` for agents.
- `biotope propose-alignment` ‚Äî propose cross-mapping `same_node` equivalences.

## Git-like metadata VCS

- `biotope status` ‚Äî show staged/modified files and validation state.
- `biotope commit` ‚Äî commit metadata changes.
- `biotope log` ‚Äî show metadata commit history.
- `biotope push` / `biotope pull` ‚Äî sync metadata with a remote.
- `biotope check-data` ‚Äî verify data files against recorded checksums.

## Knowledge-graph construction

- `biotope build` ‚Äî materialise a runnable BioCypher project from mappings + alignment. Emits `config/schema_config.yaml` (with `namespace` and autogenerated `input_label`) and per-mapping generated Python under `build/generated/<stem>/`.
- `biotope view` ‚Äî node/edge counts for the most recent build (or project competence questions if no build yet).

## Annotation + project config

- `biotope annotate` ‚Äî `apply` (merge a curated `.biotope.yaml` scaffold into a dataset's Croissant manifest, with optional `--set dataset.<field>=‚Ä¶` / `--set record_set.<field>=‚Ä¶` overrides), `edit` (interactive annotation), `load` (sample records via the manifest), `validate` (mlcroissant validation).
- `biotope config` ‚Äî manage project-level validation rules, remote validation URLs, and project metadata.

## Stubs / not yet wired

- `biotope discover` ‚Äî rank registered adapters and local Croissant files against `required_entities`. Exists as a CLI entry but the registry surface is not yet wired into the recommended workflow; the tutorial does not use it.
- `biotope benchmark` ‚Äî quality/coverage metrics. v1 stub: emits a skeleton JSON object so downstream tooling can structure-test against it. Real metric implementations land iteratively.
- `biotope read` ‚Äî NLP ingestion + health-check entry. Promise.
- `biotope search` ‚Äî registry search across MCP / biotools. Auxiliary; not used in the standard build path.

## Deprecated

- `biotope describe` ‚Äî removed; folded into `biotope map` intent flags.
- `biotope propose-mapping` ‚Äî deprecated alias for `biotope map scaffold`. The old heuristic ("one RecordSet per node type, FK fields as edges") is gone; the alias now produces an unresolved scaffold for human/agent completion.
